## مقدمه

کارمند (Staff) از طریق Web Panel به سیستم دسترسی دارد. هر کارمند:
- به یک tenant تعلق دارد
- یک یا چند role دارد (owner, manager, cashier, viewer)
- به یک یا چند شعبه دسترسی دارد

---

## SF-001: ورود به سیستم (OTP Login)

### پیش‌شرط‌ها
- کارمند توسط owner در سیستم ثبت شده است
- شماره تلفن در DB وجود دارد

### مراحل

```mermaid
flowchart TD
    A[صفحه ورود Web Panel] --> B[وارد کردن شماره تلفن]
    B --> C[کلیک ارسال کد]
    C --> D{Rate limit رعایت شده؟}
    D -->|خیر| E[خطا: بیش از ۳ درخواست در دقیقه]
    D -->|بله| F[کد OTP ارسال می‌شود به SMS]
    F --> G[وارد کردن کد ۶ رقمی]
    G --> H{کد صحیح است؟}
    H -->|خیر یا منقضی| I[خطا: کد نامعتبر است]
    H -->|بله| J[JWT صادر می‌شود]
    J --> K[دریافت لیست tenantهای فعال این کارمند]
    K --> L{چند tenant؟}
    L -->|یک tenant| M[مستقیم وارد داشبورد می‌شود]
    L -->|چند tenant| N[صفحه انتخاب tenant]
    N --> M
    M --> O[انتخاب شعبه فعال]
```

### پس‌شرط‌ها
- Access token (15 دقیقه) در حافظه
- Refresh token (30 روز) در cookie httpOnly
- شعبه فعال در session

### خطاها

| خطا | کد | HTTP |
|-----|-----|------|
| Rate limit | `OTP_RATE_LIMIT` | 429 |
| کد نامعتبر / منقضی | `OTP_INVALID_OR_EXPIRED` | 400 |
| کارمند غیرفعال | `STAFF_INACTIVE` | 403 |

## SF-002: ایجاد فروش جدید

### پیش‌شرط‌ها
- کارمند permission `installments.sale.create` دارد
- کارمند به شعبه انتخابی دسترسی دارد
- مشتری در tenant وجود دارد

### مراحل

```mermaid
flowchart TD
    A[داشبورد → فروش جدید] --> B[انتخاب مشتری]
    B --> C{مشتری وجود دارد؟}
    C -->|خیر| D[ایجاد مشتری جدید]
    C -->|بله| E[ادامه]
    D --> E
    E --> F[وارد کردن اطلاعات فروش]
    F --> G[تعیین مبلغ کل]
    G --> H[تعیین پیش‌پرداخت اختیاری]
    H --> I[تعیین تعداد اقساط]
    I --> J[تعیین تاریخ اولین قسط]
    J --> K[تعیین فاصله بین اقساط]
    K --> L[پیش‌نمایش جدول اقساط]
    L --> M{تأیید؟}
    M -->|ویرایش| F
    M -->|تأیید| N[ایجاد فروش]
    N --> O[نمایش صفحه فروش جدید]
    O --> P[اختیاری: ارسال لینک ربات به مشتری]
```

### فرم ایجاد فروش — فیلدها

| فیلد | نوع | اجباری | توضیح |
|------|-----|--------|-------|
| مشتری | جستجو | بله | جستجو با نام یا شماره |
| عنوان | text | خیر | شرح کالا/خدمت |
| مبلغ کل | عدد تومان | بله | تبدیل به ریال |
| پیش‌پرداخت | عدد تومان | خیر | پیش‌فرض: 0 |
| تعداد اقساط | عدد | بله | ۱ تا ۱۲۰ |
| تاریخ قسط اول | Datepicker | بله | Jalali |
| فاصله (روز) | عدد | بله | پیش‌فرض: 30 |
| یادداشت | textarea | خیر | |

### پیش‌نمایش جدول

قبل از ثبت نهایی، جدول کامل اقساط نمایش داده می‌شود:

```
قسط ۱ — ۱,۵۰۰,۰۰۰ تومان — ۱۴۰۵/۰۵/۰۱
قسط ۲ — ۱,۵۰۰,۰۰۰ تومان — ۱۴۰۵/۰۶/۰۱
قسط ۳ — ۱,۵۰۰,۰۰۰ تومان — ۱۴۰۵/۰۷/۰۱
قسط ۴ — ۱,۵۰۰,۰۰۰ تومان — ۱۴۰۵/۰۸/۰۱
─────────────────────────────────
جمع: ۶,۰۰۰,۰۰۰ تومان
پیش‌پرداخت: ۰
کل فروش: ۶,۰۰۰,۰۰۰ تومان ✓
```

### پس‌شرط‌ها
- Sale با status `active` در DB
- N تا Installment با status `pending`
- AuditLog (`sale.create`)
- OutboxEvent (`SaleCreated`)

## SF-003: مشاهده و مدیریت فروش‌ها

### مراحل

```mermaid
flowchart TD
    A[منو → فروش‌ها] --> B[لیست فروش‌ها با فیلتر]
    B --> C{فیلترها}
    C -->|status| D[active / completed / cancelled]
    C -->|مشتری| E[جستجو با نام/شماره]
    C -->|تاریخ| F[بازه ایجاد]
    C -->|شعبه| G[بر اساس شعبه فعال]
    D --> H[کلیک روی فروش]
    E --> H
    F --> H
    H --> I[صفحه جزئیات فروش]
    I --> J[جدول اقساط]
    I --> K[تاریخچه پرداخت‌ها]
```

| بازه تاریخ | date range | Jalali |
| شعبه | dropdown | بر اساس دسترسی کارمند |
| مبلغ | range | حداقل/حداکثر |

## SF-004: تأیید پرداخت

### پیش‌شرط‌ها
- کارمند permission `installments.payment.confirm` دارد
- `PaymentAttempt` با status `pending` وجود دارد

### مراحل

```mermaid
flowchart TD
    A[اعلان: پرداخت جدید ثبت شد] --> B[صفحه تأیید پرداخت]
    B --> C[نمایش اطلاعات: مشتری، مبلغ، رسید]
    C --> D[مشاهده تصویر رسید اختیاری]
    D --> E{تصمیم}
    E -->|تأیید| F[کلیک تأیید]
    E -->|رد| G[وارد کردن دلیل رد]
    F --> H[PaymentAttempt → confirmed]
    F --> I[Installment → paid]
    F --> J[AuditLog + OutboxEvent]
    J --> K[اعلان به مشتری]
    G --> L[PaymentAttempt → rejected]
    G --> M[AuditLog + OutboxEvent]
    M --> N[اعلان به مشتری با دلیل رد]
```

### صفحه تأیید پرداخت

```
─────────────────────────────────────────
تأیید پرداخت — قسط ۳ از ۴
─────────────────────────────────────────
مشتری: علی محمدی
فروش: #SALE-1234 — گوشی موبایل
مبلغ قسط: ۱,۵۰۰,۰۰۰ تومان

گزارش مشتری در:
  مبلغ گزارش‌شده: ۱,۵۰۰,۰۰۰ تومان
  یادداشت: «واریز کردم به حساب شما»
  رسید: [مشاهده تصویر]

[✅ تأیید پرداخت]  [❌ رد پرداخت]
```

### خطاها

| خطا | کد |
|-----|-----|
| پرداخت قبلاً تأیید شده | `PAYMENT_ALREADY_CONFIRMED` |
| پرداخت قبلاً رد شده | `PAYMENT_ALREADY_REJECTED` |
| قسط قبلاً paid | `INSTALLMENT_ALREADY_PAID` |

## SF-005: بخشودگی قسط (Waive)

### پیش‌شرط‌ها
- کارمند permission `installments.installment.waive` دارد
- قسط در وضعیت `pending` یا `overdue` است

### مراحل

```mermaid
flowchart TD
    A[صفحه فروش → قسط] --> B[دکمه بخشودگی]
    B --> C[نمایش هشدار تأییدیه]
    C --> D[وارد کردن دلیل بخشودگی اجباری]
    D --> E{تأیید نهایی؟}
    E -->|خیر| F[لغو]
    E -->|بله| G[Installment → waived]
    G --> H[AuditLog با دلیل]
    H --> I[بررسی completion فروش BR-018]
    I --> J{همه settled؟}
    J -->|بله| K[Sale → completed]
    J -->|خیر| L[ادامه]
```

### خطاها

| خطا | کد |
|-----|-----|
| قسط paid | `INSTALLMENT_ALREADY_PAID` |
| قسط waived | `INSTALLMENT_ALREADY_WAIVED` |
| permission ندارد | `PERMISSION_DENIED` |

## SF-006: لغو فروش

### پیش‌شرط‌ها
- کارمند permission `installments.sale.cancel` دارد
- فروش `active` است
- هیچ قسط `paid` وجود ندارد (BR-012)

### مراحل

```mermaid
flowchart TD
    A[صفحه فروش] --> B[دکمه لغو فروش]
    B --> C[چک: هیچ قسط paid ندارد؟]
    C -->|دارد| D[خطا: فروش با قسط پرداخت‌شده لغو نمی‌شود]
    C -->|ندارد| E[نمایش دیالوگ تأیید]
    E --> F[وارد کردن دلیل لغو اجباری]
    F --> G[تأیید نهایی]
    G --> H[Sale → cancelled]
    H --> I[AuditLog با دلیل]
    I --> J[اعلان به مشتری اختیاری]
```

## SF-007: مدیریت مشتریان

### ۷.۱ ثبت مشتری جدید

```mermaid
flowchart TD
    A[منو → مشتریان → جدید] --> B[وارد کردن اطلاعات]
    B --> C[شماره تلفن اجباری]
    C --> D[normalize: 09xxxxxxxxx]
    D --> E{مشتری با این شماره وجود دارد؟}
    E -->|بله در tenant| F[نمایش اطلاعات موجود + ویرایش اختیاری]
    E -->|بله در platform نه tenant| G[TenantCustomer ایجاد می‌شود — upsert]
    E -->|خیر| H[User + GlobalCustomer + TenantCustomer ایجاد می‌شود]
    F --> I[ذخیره]
    G --> I
    H --> I
```

### فیلدهای مشتری

| فیلد | اجباری | توضیح |
|------|--------|-------|
| شماره تلفن | بله | normalize به 09xxxxxxxxx |
| نام | بله | حداقل ۲ حرف |
| کد محلی | خیر | کد داخلی tenant |
| یادداشت | خیر | |

### ۷.۲ Import Excel

```mermaid
flowchart TD
    A[منو → مشتریان → Import] --> B[دانلود template Excel]
    B --> C[پر کردن فایل]
    C --> D[آپلود فایل]
    D --> E[validation هر row]
    E --> F{row معتبر؟}
    F -->|بله| G[User موجود → GlobalCustomer + TenantCustomer link]
    F -->|خیر| H[skip + ثبت در errors]
    G --> I[نمایش نتیجه نهایی]
    H --> I
    I --> J[totalRows, successCount, failedCount]
    I --> K[دانلود فایل خطاها اختیاری]
```

## SF-009: تنظیمات یادآور

### پیش‌شرط‌ها
- کارمند permission `installments.reminder.configure` دارد

### مراحل

```mermaid
flowchart TD
    A[تنظیمات → یادآورها] --> B[نمایش تنظیمات فعلی]
```
