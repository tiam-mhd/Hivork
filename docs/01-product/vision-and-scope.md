# چشم‌انداز و Scope محصول

## Hivork چیست؟

**Hivork** یک پلتفرم SaaS **ماژولار** برای کسب‌وکارهای خرده‌فروشی ایران است که از یک **هسته مشترک** (tenant، RBAC، تنظیمات، notification) تغذیه می‌شود و **ماژول‌های افزایشی** روی آن سوار می‌شوند.

### ماژول اول: مدیریت اقساط

- فروشنده فروش قسطی ثبت می‌کند
- مشتری یادآور دریافت می‌کند (ربات تلگرام/بله)
- مشتری پرداخت را ثبت می‌کند؛ فروشنده تأیید می‌کند
- تقویم اقساط و گزارش معوقات
- مشتری اقساط شخصی خود را هم مدیریت می‌کند

### چشم‌انداز ۱۰ ساله

| امروز | فردا |
|-------|------|
| یادآور و ثبت اقساط | زیرساخت اعتماد و نقدینگی خرده‌فروشی |
| RBAC + multi-tenant | پلتفرم white-label |
| ربات + وب | API برای POS / حسابداری |
| — | امتیاز اعتباری، پرداخت، analytics |

## Actors (بازیگران سیستم)

| Actor | توضیح | کانال اصلی |
|-------|--------|------------|
| **Platform Admin** | تیم Hivork، پشتیبانی، مدیریت tenant | پنل platform |
| **Staff** | صاحب/کارمند فروشگاه (tenant user) | پنل وب، ربات فروشنده |
| **Global Customer** | مشتری نهایی (هویت سراسری با موبایل) | ربات، PWA مشتری |
| **System/Bot** | service account داخلی (scheduler، webhook) | internal |

> **تأیید شده (ADR-017):** یک شماره موبایل = یک `User` platform. `GlobalCustomer` پروفایل B2C اختیاری (1:1) است.  
> Staff و Customer می‌توانند همان User/phone داشته باشند — دو actor جدا با token/session جدا (ADR-011, ADR-017).

## Scope ماژول اقساط (In / Out)

### In Scope — فاز ۱–۳

- ثبت tenant، branch، staff
- RBAC کامل (role system + custom)
- CRUD مشتری tenant (`TenantCustomer`)
- ثبت فروش + تقسیم اقساط
- یادآور خودکار (تلگرام، بعداً بله)
- ثبت پرداخت توسط مشتری + تأیید فروشنده
- داشبورد معوقات و سررسید امروز
- پورتال مشتری (تقویم شمسی، تاریخچه)
- اقساط شخصی مشتری (`PersonalInstallment`)
- Import مشتری از Excel
- Audit log عملیات حساس
- Tenant settings (reminder rules، ...)

### Out of Scope — فعلاً

- درگاه پرداخت آنلاین
- فاکتور رسمی / حسابداری کامل
- اپ Native iOS/Android
- marketplace چندفروشنده برای مشتری
- AI/chatbot
- امتیاز اعتباری B2B marketplace

## ارزش پیشنهادی (Value Proposition)

### برای فروشنده

> «امروز کی باید پیگیری کنم؟ چقدر معوق دارم؟ کدام مشتری اعتبارش خراب شده؟»

ابزار **مدیریت نقدینگی**، نه فقط تقویم یادآور.

### برای مشتری

> یادآور بدون نصب اپ — در تلگرام/بله.  
> تقویم کامل در PWA.

## ماژول‌های آینده (پس از اقساط)

| ماژول | توضیح کوتاه |
|-------|-------------|
| منوی دیجیتال | طراحی منو + سفارش (پروژه دوم — فاز بعد) |
| POS / صندوق | اتصال به فروش |
| CRM | پیگیری مشتری |
| Analytics | گزارش‌های پیشرفته |
| Payment | درگاه و تسویه |

هر ماژول: permissions، settings schema، API، UI routes، events، jobs — از طریق **Module Registry**.

---

## اصول طراحی محصول

1. **Business Logic فقط در Backend Domain** — همه کانال‌ها client
2. **Multi-tenant از روز ۱** — isolation اجباری
3. **Configurable ولی کنترل‌شده** — settings schema، نه free-form rules
4. **Audit همه عملیات مالی**
5. **تقویم شمسی + timezone Asia/Tehran** در همه لایه‌ها
6. **مبلغ به ریال (BIGINT)** — نمایش تومان در UI
